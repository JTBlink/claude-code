# 解释性输出风格插件

> 重新创建已弃用的解释性输出风格作为 SessionStart 钩子。

## 概述

此插件在每个会话开始时自动添加指令，鼓励 Claude 提供教育性的实现选择见解、代码库模式和决策解释。

## 工作原理

插件使用 SessionStart 钩子向每个会话注入额外上下文。此上下文指示 Claude 在编写代码前后提供简要的教育性解释，格式如下：

```
`★ 洞察 ─────────────────────────────────────`
[2-3 个关键教育要点]
`─────────────────────────────────────────────────`
```

## 功能特点

当启用时，此插件自动添加指令，鼓励 Claude：

1. 提供关于实现选择的教育性见解
2. 解释代码库模式和决策
3. 平衡任务完成与学习机会

## 洞察内容

洞察内容专注于：

- 针对代码库的具体实现选择
- 代码中的模式和约定
- 权衡和设计决策
- 代码库特定细节而非通用编程概念

## 使用方式

安装后，插件在每个会话开始时自动激活。无需额外配置。

## 从输出风格迁移

此插件替代已弃用的"解释性"(Explanatory)输出风格设置。如果你之前使用：

```json
{
  "outputStyle": "Explanatory"
}
```

现在可以通过安装此插件来实现相同的行为。

更一般地说，这种 SessionStart 钩子模式大致等同于 CLAUDE.md，但它更灵活，并允许通过插件分发。

**注意**: 涉及软件开发以外任务的输出风格，最好以 [子代理](https://docs.claude.com/en/docs/claude-code/sub-agents) 的形式表达，而不是 SessionStart 钩子。子代理更改系统提示，而 SessionStart 钩子添加到默认系统提示。

## 警告

**请勿安装此插件**，除非你愿意承担此插件额外指令和输出的 token 成本。

## 管理更改

- **禁用插件** - 保留代码安装在设备上
- **卸载插件** - 从设备删除代码
- **更新插件** - 创建此插件的本地副本以个性化
  - 提示: 让 Claude 阅读 https://docs.claude.com/en/docs/claude-code/plugins.md 并为你设置
