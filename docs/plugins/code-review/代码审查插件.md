# code-review 插件

> 多 Agent 并行代码审查，自动化 PR 审查流程

## 功能说明

`code-review` 插件使用 5 个 specialized agents 并行审查代码，通过置信度评分过滤误报，提供高质量的审查反馈。

## 安装方式

```bash
# 插件已包含在项目中
# 直接在会话中使用 /code-review 命令
```

## 命令参考

### `/code-review [选项]`

启动自动化代码审查流程。

**使用方式**:
```bash
# 标准审查
/code-review

# 添加评论到 PR
/code-review --comment

# 调整置信度阈值
/code-review --confidence 90

# 指定审查范围
/code-review --files src/auth/*.py
```

**选项**:
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--comment` | 在 PR 中添加评论 | false |
| `--confidence <阈值>` | 置信度阈值 (0-100) | 80 |
| `--files <模式>` | 指定审查文件 | 全部 |
| `--verbose` | 详细输出 | false |

## 审查 Agent

插件启动 5 个并行 Agent，每个专注于不同的审查方面：

### Agent 1: CLAUDE.md 合规检查

**职责**: 检查代码是否符合项目规范

**检查项**:
- 命名规范（camelCase/snake_case）
- 格式化（缩进、行宽）
- 代码结构（函数大小、模块组织）
- 文档注释

**输出示例**:
```markdown
## CLAUDE.md 合规性

❌ **问题**: 函数体超过 50 行
📍 位置：src/auth/login.py:45
📋 建议：拆分为多个小函数
```

---

### Agent 2: Bug 扫描

**职责**: 检测明显的 Bug 和代码异味

**检查项**:
- 空指针/未null 检查
- 资源泄漏（文件、连接）
- 边界条件
- 类型错误
- 逻辑错误

**输出示例**:
```markdown
## Bug 检测

🔴 **严重**: 未处理空值
📍 位置：src/user.py:23
📋 问题：user.get_name() 可能返回 None
🔧 修复：添加 null 检查
```

---

### Agent 3: 历史上下文分析

**职责**: 分析 Git 历史理解变更背景

**检查项**:
- 相关文件的历史变更
- 之前的 Bug 修复
- 架构决策记录
- 技术债务

**输出示例**:
```markdown
## 历史上下文

📜 **相关历史**:
- 2024-01-15: 修复认证 Bug (#42)
- 2024-02-20: 重构用户模型 (#58)

⚠️ **注意**: 此区域之前出现过 Bug，请仔细测试
```

---

### Agent 4: PR 历史分析

**职责**: 查看 PR 的评论和变更历史

**检查项**:
- 之前的审查评论
- 已修复的问题
- 待处理的讨论
- 变更演进

**输出示例**:
```markdown
## PR 历史

📝 **之前评论**:
- # 关于错误处理的讨论（已解决）
- # 关于性能的担忧（待验证）

✅ **已修复**:
- 初始 Bug 已修复
- 测试已添加
```

---

### Agent 5: 代码注释检查

**职责**: 检查代码注释和文档

**检查项**:
- 公共 API 文档
- 复杂逻辑解释
- TODO/FIXME 标记
- 过时注释

**输出示例**:
```markdown
## 注释质量

⚠️ **缺失文档**:
- 公共函数 `authenticate()` 缺少 docstring

📝 **过时注释**:
- 第 45 行：注释提到已删除的函数

✅ **良好实践**:
- 复杂算法有详细解释
```

## 审查流程

### 步骤 1: 启动 Agent

```python
# 并行启动 5 个 Agent
agents = [
    CLAUDEmdComplianceAgent(),
    BugScannerAgent(),
    HistoricalContextAgent(),
    PRHistoryAgent(),
    CodeCommentsAgent()
]

results = await asyncio.gather(*[agent.run(pr) for agent in agents])
```

### 步骤 2: 收集反馈

每个 Agent 返回结构化反馈：

```json
{
  "agent": "bug-scanner",
  "findings": [
    {
      "severity": "critical",
      "location": "src/auth.py:45",
      "description": "未处理空值",
      "suggestion": "添加 null 检查",
      "confidence": 85
    }
  ]
}
```

### 步骤 3: 置信度评分

计算每个发现的置信度：

```python
confidence_score = (
    pattern_match_score * 0.4 +
    context_score * 0.3 +
    historical_score * 0.2 +
    code_quality_score * 0.1
)
```

### 步骤 4: 过滤误报

过滤掉置信度低于阈值的发现：

```python
filtered = [f for f in findings if f.confidence >= threshold]
```

### 步骤 5: 生成报告

整合所有 Agent 的反馈：

```markdown
# 代码审查报告

## 严重问题 (2)
...

## 主要问题 (5)
...

## 建议 (8)
...
```

### 步骤 6: 添加评论（可选）

如果指定 `--comment`，在 PR 中添加评论：

```python
if add_comments:
    for finding in filtered:
        gh.pr.comment(pr_number, format_comment(finding))
```

## 置信度评分

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **模式匹配** | 40% | 问题模式匹配程度 |
| **上下文** | 30% | 与代码上下文的一致性 |
| **历史** | 20% | 历史数据支持度 |
| **代码质量** | 10% | 代码质量指标 |

### 置信度级别

| 级别 | 分数 | 处理建议 |
|------|------|----------|
| **确定** | 90-100 | 必须修复 |
| **高** | 80-89 | 应该修复 |
| **中** | 70-79 | 建议修复 |
| **低** | 60-69 | 可选修复 |
| **很低** | <60 | 可能误报 |

## 配置选项

### 自定义配置

在项目根目录创建 `.code-review.json`:

```json
{
  "threshold": 80,
  "agents": {
    "claude-md": {
      "enabled": true,
      "configFile": "CLAUDE.md"
    },
    "bug-scanner": {
      "enabled": true,
      "severityLevels": ["critical", "major", "minor"]
    },
    "historical-context": {
      "enabled": true,
      "lookbackCommits": 50
    },
    "pr-history": {
      "enabled": true
    },
    "code-comments": {
      "enabled": true,
      "requireDocstring": true
    }
  },
  "output": {
    "format": "markdown",
    "addComments": false,
    "verbose": false
  }
}
```

### GitHub Actions 配置

```yaml
# .github/workflows/code-review.yml
name: Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Run Code Review
        uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: |
            /code-review --comment --confidence 80
```

## 最佳实践

### 审查时机

- **PR 创建后**: 自动触发审查
- **重大变更后**: 手动触发审查
- **发布前**: 全面审查

### 阈值选择

| 项目类型 | 推荐阈值 |
|----------|----------|
| 生产代码 | 85-90 |
| 测试代码 | 75-80 |
| 原型/实验 | 60-70 |

### 反馈处理

1. **优先处理严重问题**: 安全漏洞、功能 Bug
2. **评估主要问题**: 代码质量、可维护性
3. **选择性处理建议**: 风格、优化

### 避免误报

- 调整阈值过滤低置信度发现
- 结合项目实际情况判断
- 必要时人工复核

## 故障排查

### 问题：审查时间过长

**原因**: 代码量大或 Agent 过多

**解决方案**:
```bash
# 指定审查文件
/code-review --files src/auth/*.py

# 禁用部分 Agent
# 编辑 .code-review.json
{
  "agents": {
    "pr-history": { "enabled": false }
  }
}
```

### 问题：误报过多

**原因**: 阈值过低

**解决方案**:
```bash
# 提高阈值
/code-review --confidence 90
```

### 问题：评论添加失败

**原因**: 权限不足或 API 限制

**解决方案**:
```bash
# 检查权限
gh repo view

# 手动添加评论
/code-review > review.md
# 然后手动复制内容到 PR
```
