# 提交命令插件

> 简化 Git 工作流，一键完成提交、推送和创建 Pull Request。

## 概述

提交命令插件自动化常见的 Git 操作，减少上下文切换和手动命令执行。使用单个斜杠命令处理整个工作流程。

## 命令

### `/commit`

基于已暂存和未暂存的更改，自动生成提交信息并创建 Git 提交。

**执行流程**:
1. 分析当前 Git 状态
2. 审查已暂存和未暂存的更改
3. 检查最近的提交信息以匹配仓库风格
4. 起草适当的提交信息
5. 暂存相关文件
6. 创建提交

**使用方式**:
```bash
/commit
```

**特性**:
- 自动起草匹配仓库风格的提交信息
- 遵循 Conventional Commits 规范
- 避免提交包含密钥的文件（.env、credentials.json）
- 提交信息中包含 Claude Code 归属

---

### `/commit-push-pr`

完整工作流命令，一步完成提交、推送和创建 Pull Request。

**执行流程**:
1. 创建新分支（如果当前在 main 上）
2. 暂存并提交更改
3. 推送分支到 origin
4. 使用 `gh pr create` 创建 Pull Request
5. 提供 PR URL

**使用方式**:
```bash
/commit-push-pr
```

**特性**:
- 分析分支中的所有提交（不只是最新的）
- 创建全面的 PR 描述:
  - 变更摘要（1-3 个要点）
  - 测试计划检查清单
  - Claude Code 归属
- 自动处理分支创建
- 使用 GitHub CLI (`gh`) 创建 PR

**要求**:
- GitHub CLI (`gh`) 必须安装并认证
- 仓库必须有名为 `origin` 的远程

---

### `/clean_gone`

清理已从远程仓库删除的本地分支。

**执行流程**:
1. 列出所有本地分支以识别 [gone] 状态
2. 识别并删除与 [gone] 分支关联的 worktree
3. 删除所有标记为 [gone] 的分支
4. 提供已删除分支的反馈

**使用方式**:
```bash
/clean_gone
```

**特性**:
- 处理常规分支和 worktree 分支
- 删除分支前安全移除 worktree
- 显示清晰的删除反馈
- 报告是否需要清理

**使用时机**:
- PR 合并并删除远程分支后
- 本地分支列表充斥着过时分支时
- 定期仓库维护期间

## 最佳实践

### 使用 `/commit`
- 提交前审查暂存的更改
- 让 Claude 分析你的更改并匹配仓库提交风格
- 信任自动化消息，但验证其准确性
- 用于开发期间的常规提交

### 使用 `/commit-push-pr`
- 准备创建 PR 时使用
- 确保所有更改已完成并测试
- Claude 会分析完整分支历史来生成 PR 描述
- 审查 PR 描述，必要时进行编辑
- 用于减少上下文切换

### 使用 `/clean_gone`
- 定期运行以保持分支列表整洁
- 在合并多个 PR 后特别有用
- 安全运行 - 只删除远程已删除的分支
- 帮助维护整洁的本地仓库

## 工作流集成

### 快速提交流程
```bash
# 编写代码
/commit
# 继续开发
```

### 功能分支工作流
```bash
# 开发功能，多次提交
/commit  # 第一次提交
# 更多更改
/commit  # 第二次提交
# 准备创建 PR
/commit-push-pr
```

### 维护工作流
```bash
# 多个 PR 合并后
/clean_gone
# 清理的工作区准备下一个功能
```

## 要求

- Git 必须安装并配置
- `/commit-push-pr` 需要: GitHub CLI (`gh`) 安装并认证
- 仓库必须是 Git 仓库且有远程

## 故障排除

### `/commit` 创建空提交

**问题**: 没有更改可提交

**解决方案**:
- 确保有未暂存或已暂存的更改
- 运行 `git status` 验证更改存在

### `/commit-push-pr` 无法创建 PR

**问题**: `gh pr create` 命令失败

**解决方案**:
- 安装 GitHub CLI: `brew install gh` (macOS) 或查看 [GitHub CLI 安装](https://cli.github.com/)
- 认证: `gh auth login`
- 确保仓库有 GitHub 远程

### `/clean_gone` 找不到分支

**问题**: 没有标记为 [gone] 的分支

**解决方案**:
- 运行 `git fetch --prune` 更新远程跟踪
- 分支必须从远程删除才会显示为 [gone]

---

## 提示

- **与其他工具结合**: 开发期间使用 `/commit`，准备好后用 `/commit-push-pr`
- **让 Claude 起草消息**: 提交信息分析会学习你仓库的风格
- **定期清理**: 每周运行 `/clean_gone` 维护整洁的分支列表
- **推送前审查**: 始终审查提交消息和更改后再推送
