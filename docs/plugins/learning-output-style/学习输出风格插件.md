# 学习输出风格插件

> 结合交互式学习模式和教育性见解作为 SessionStart 钩子。

## 概述

此插件将未发布的 Learning 输出风格与解释性功能结合为 SessionStart 钩子。

**注意**: 此插件与原始未发布的 Learning 输出风格不同，它还包含了 [explanatory-output-style 插件](https://github.com/anthropics/claude-code/tree/main/plugins/explanatory-output-style) 的所有功能，提供交互式学习和教育性见解。

## 功能特点

当启用时，此插件自动添加指令，鼓励 Claude：

1. **学习模式**: 通过在决策点请求你有意义的代码贡献来让你主动学习
2. **解释模式**: 提供关于实现选择和代码库模式的教育性见解

Claude 不会自动实现所有内容，而是：

1. 识别你可以编写 5-10 行有意义代码的机会
2. 专注于你的输入真正重要的业务逻辑和设计选择
3. 准备上下文和位置供你贡献
4. 解释权衡并指导你的实现
5. 在编写代码前后提供教育性见解

## 工作原理

插件使用 SessionStart 钩子向每个会话注入额外上下文。此上下文指示 Claude 采用交互式教学方法，让你主动参与编写代码的关键部分。

## Claude 请求贡献的时机

Claude 会要求你编写以下内容的代码：

- 有多种有效方法的业务逻辑
- 错误处理策略
- 算法实现选择
- 数据结构决策
- 用户体验决策
- 设计模式和架构选择

## Claude 不请求贡献的时机

Claude 会直接实现：

- 样板或重复代码
- 没有有意义选择的明显实现
- 配置或设置代码
- 简单的 CRUD 操作

## 示例交互

**Claude**: 我已经设置了认证中间件。会话超时行为是安全性与用户体验的权衡 - 会话应该在活动时自动延长，还是有硬性超时？

在 `auth/middleware.ts` 中，实现 `handleSessionTimeout()` 函数来定义超时行为。

考虑：自动延长改善用户体验，但可能让会话开放更长时间；硬性超时更安全，但可能会让活跃用户感到沮丧。

**你**: [编写 5-10 行实现你首选方法]

## 教育性见解

除了交互式学习，Claude 还会使用以下格式提供关于实现选择的教育性见解：

```
`★ 洞察 ─────────────────────────────────────`
[2-3 个关于代码库或实现的关键教育要点]
`─────────────────────────────────────────────────`
```

这些见解专注于：

- 针对代码库的具体实现选择
- 代码中的模式和约定
- 权衡和设计决策
- 代码库特定细节而非通用编程概念

## 使用方式

安装后，插件在每个会话开始时自动激活。无需额外配置。

## 从输出风格迁移

此插件结合了未发布的"学习"(Learning) 输出风格和已弃用的"解释性"(Explanatory) 输出风格。它提供交互式学习体验，让你在有意义的决策点主动贡献代码，同时接收关于实现选择的教育性见解。

如果你之前使用 explanatory-output-style 插件，此学习插件包含所有该功能外加交互式学习特性。

这种 SessionStart 钩子模式大致等同于 CLAUDE.md，但它更灵活，并允许通过插件分发。

## 警告

**请勿安装此插件**，除非你愿意承担此插件额外指令的 token 成本和学习模式的交互性质。

## 管理更改

- **禁用插件** - 保留代码安装在设备上
- **卸载插件** - 从设备删除代码
- **更新插件** - 创建此插件的本地副本以个性化
  - 提示: 让 Claude 阅读 https://docs.claude.com/en/docs/claude-code/plugins.md 并为你设置

## 哲学

边做边学比被动观察更有效。此插件将你与 Claude 的互动从"观看学习"转变为"构建理解"，确保你通过有意义逻辑的动手编码发展实践技能。
