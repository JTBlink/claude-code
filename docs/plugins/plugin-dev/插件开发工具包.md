# plugin-dev 插件

> 插件开发完整工具包，提供 7 个专家技能和 AI 辅助创建

## 功能说明

`plugin-dev` 是一个全面的插件开发工具包，提供 8 阶段引导式创建流程和 7 个专家技能，帮助开发者快速创建高质量的 Claude Code 插件。

## 安装方式

```bash
# 插件已包含在项目中
# 直接在会话中使用 /plugin-dev:create-plugin 命令
```

## 命令参考

### `/plugin-dev:create-plugin`

启动 8 阶段引导式插件创建流程。

**使用方式**:
```bash
/plugin-dev:create-plugin
```

**交互流程**:
1. 插件信息收集
2. 目录结构创建
3. 插件元数据配置
4. 命令开发（可选）
5. Agent 开发（可选）
6. Skill 开发（可选）
7. Hook 开发（可选）
8. 验证与文档

---

## 8 阶段创建流程

### 阶段 1: 插件信息收集

**目标**: 明确插件的功能和范围

**问题示例**:
```
让我帮你创建一个新插件！请告诉我：

1. 插件名称：
   （例如：my-custom-plugin）

2. 插件描述：
   （简短描述插件的功能）

3. 主要功能：
   - [ ] 自定义命令
   - [ ] 专用 Agent
   - [ ] Agent 技能
   - [ ] 事件 Hooks
   - [ ] MCP 集成

4. 目标用户：
   （谁会使用这个插件）
```

**输出**:
```json
{
  "name": "my-custom-plugin",
  "description": "自定义功能插件",
  "features": ["commands", "hooks"],
  "targetUsers": "开发团队"
}
```

---

### 阶段 2: 目录结构创建

**目标**: 创建标准插件目录结构

**创建的文件**:
```
my-custom-plugin/
├── .claude-plugin/
│   └── plugin.json          # 插件元数据
├── commands/                # 斜杠命令
│   └── .gitkeep
├── agents/                  # 专用 Agent
│   └── .gitkeep
├── skills/                  # Agent 技能
│   └── .gitkeep
├── hooks/                   # 事件钩子
│   └── .gitkeep
├── .mcp.json                # MCP 配置
└── README.md                # 插件文档
```

**命令**:
```bash
mkdir -p my-custom-plugin/{.claude-plugin,commands,agents,skills,hooks}
```

---

### 阶段 3: 插件元数据配置

**目标**: 配置插件基本信息

**生成的 plugin.json**:
```json
{
  "name": "my-custom-plugin",
  "version": "1.0.0",
  "description": "自定义功能插件",
  "author": {
    "name": "Your Name",
    "email": "your.email@example.com"
  },
  "features": {
    "commands": true,
    "agents": false,
    "skills": false,
    "hooks": true
  },
  "claudeCodeVersion": ">=1.0.0"
}
```

**字段说明**:
| 字段 | 说明 | 必填 |
|------|------|------|
| `name` | 插件唯一标识 | ✅ |
| `version` | 版本号（语义化） | ✅ |
| `description` | 简短描述 | ✅ |
| `author` | 作者信息 | ✅ |
| `features` | 启用的功能 | ❌ |
| `claudeCodeVersion` | 最低版本要求 | ❌ |

1. 解析参数
2. 执行主要逻辑
3. 返回结果

## 示例

### 阶段 7: Hook 开发（可选）

**目标**: 创建事件钩子

**使用 Skill**: `hook-development`

**Hook 模板**:
```json
{
  "hooks": {
    "<事件类型>": [
      {
        "matcher": "<匹配器>",
        "hooks": [
          {
            "type": "command",
            "command": "<执行命令>"
          }
        ]
      }
    ]
  }
}
```

**事件类型**:
- `PreToolUse` - 工具调用前
- `PostToolUse` - 工具调用后
- `SessionStart` - 会话开始时
- `SessionEnd` - 会话结束时
- `Stop` - Agent 想要停止时
- `Prompt` - 用户提交提示时

**详细文档**: [hook-development/README.md](./skills/hook-development/README.md)

```

**生成的 README.md**:

# My Custom Plugin

> 插件简短描述

```

### 命令

```bash

```bash

```bash
# 本地开发
claude --plugin-dir ./my-custom-plugin
```
```

## 最佳实践

### 命名规范

- **插件名**: 小写，连字符分隔（`my-plugin`）
- **命令**: 简短描述性（`/deploy`）
- **Agent**: 功能 + 角色（`code-reviewer`）
- **Skill**: 动宾结构（`security-check`）

### 目录组织

```
plugin/
├── .claude-plugin/     # 元数据
├── commands/           # 命令定义
├── agents/             # Agent 定义
├── skills/             # Skill 定义
├── hooks/              # Hook 配置
├── scripts/            # 辅助脚本
└── README.md           # 文档
```

### 文档质量

- 清晰的功能说明
- 完整的使用示例
- 详细的配置说明
- 故障排查指南

### 测试策略

- 单元测试命令逻辑
- 集成测试 Hook 触发
- 端到端测试完整流程
