# ralph-wiggum 插件

> 自主迭代循环开发，Claude 重复执行任务直到完成

## 功能说明

`ralph-wiggum` 插件通过拦截 Claude 的退出尝试，实现自主迭代循环开发。Claude 会重复执行同一任务，每次迭代改进，直到满足完成条件。

## 安装方式

```bash
# 插件已包含在项目中
# 直接在会话中使用 /ralph-loop 命令
```

## 命令参考

### `/ralph-loop <任务>`

启动自主迭代循环。

**使用方式**:
```bash
# 启动循环
/ralph-loop 重构这个模块，提高代码质量

# 指定迭代次数
/ralph-loop 优化性能 --iterations 5

# 指定完成条件
/ralph-loop 添加测试 --until "测试覆盖率>90%"
```

**参数**:
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--iterations <次数>` | 最大迭代次数 | 10 |
| `--until <条件>` | 完成条件 | 自动判断 |
| `--timeout <分钟>` | 超时时间 | 60 |

---

### `/cancel-ralph`

停止自主迭代循环。

**使用方式**:
```bash
/cancel-ralph
```

**效果**:
- 停止当前迭代
- 输出当前进度
- 清理循环状态

## 工作原理

### 循环机制

```
┌─────────────────────────────────────┐
│           启动循环                   │
│  /ralph-loop <任务>                 │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│         执行任务                     │
│  Claude 执行用户指定的任务           │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│        检查完成条件                  │
│  - 达到最大迭代次数？                │
│  - 满足完成条件？                    │
│  - 超时？                           │
└─────────────┬───────────────────────┘
              │
       ┌──────┴──────┐
       │             │
      是            否
       │             │
       ▼             ▼
┌──────────┐   ┌─────────────────────┐
│  完成    │   │   拦截退出尝试      │
│  输出    │   │   Hook: Stop        │
│  总结    │   │   继续下一次迭代    │
└──────────┘   └─────────┬───────────┘
                         │
                         │ 继续
                         └──────────────┐
                                        │
                                        ▼
                              ┌─────────────────┐
                              │   评估当前状态   │
                              │   确定改进方向   │
                              └────────┬────────┘
                                       │
                                       ▼
                              （回到执行任务）
```

### Hook 拦截

**位置**: `hooks/stop-hook.sh`

```bash
#!/bin/bash
# 拦截 Stop 事件，继续迭代

input=$(cat)
decision=$(echo "$input" | jq -r '.decision')

if [ "$decision" = "stop" ]; then
    # 检查是否应该继续
    iteration=$(get_current_iteration)
    max_iterations=$(get_max_iterations)
    
    if [ "$iteration" -lt "$max_iterations" ]; then
        echo '{"action": "continue", "reason": "迭代未完成"}'
        exit 0
    fi
fi

# 允许正常退出
echo '{"action": "allow"}'
```

## 配置选项

### 自定义配置

在项目根目录创建 `.ralph-wiggum.json`:

```json
{
  "defaultIterations": 10,
  "maxIterations": 20,
  "timeout": 3600,
  "autoSave": true,
  "checkpointInterval": 3,
  "completionConditions": {
    "codeQuality": {
      "metric": "complexity",
      "threshold": 10
    },
    "testCoverage": {
      "metric": "coverage",
      "threshold": 90
    },
    "documentation": {
      "metric": "completeness",
      "threshold": 80
    }
  }
}
```

### 配置说明

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `defaultIterations` | 默认迭代次数 | 10 |
| `maxIterations` | 最大迭代次数 | 20 |
| `timeout` | 超时时间（秒） | 3600 |
| `autoSave` | 自动保存进度 | true |
| `checkpointInterval` | 检查点间隔 | 3 |

## 最佳实践

### 任务选择

**适合的任务**:
- 需要多次迭代改进
- 有明确的质量标准
- 可以渐进式完善

**不适合的任务**:
- 一次性完成的任务
- 需要人工决策的任务
- 有破坏性风险的任务

### 迭代次数

| 任务类型 | 推荐次数 |
|----------|----------|
| 代码重构 | 5-10 |
| 测试编写 | 5-15 |
| 文档完善 | 5-10 |
| 性能优化 | 3-5 |
| Bug 修复 | 3-5 |

### 完成条件

**好的条件**:
- 可量化（覆盖率>90%）
- 可验证（测试通过）
- 明确清晰（无 TODO）

**不好的条件**:
- 模糊不清（"尽可能好"）
- 无法验证（"看起来不错"）
- 主观判断（"我觉得可以"）

### 中断策略

**何时中断**:
- 发现方向错误
- 遇到技术障碍
- 需要人工决策
- 达到时间限制

**如何中断**:
```bash
/cancel-ralph
```

## 故障排查

### 问题：循环无法停止

**原因**: Hook 配置错误

**解决方案**:
```bash
# 检查 Hook 文件
cat hooks/stop-hook.sh

# 临时禁用 Hook
# 编辑 .claude/settings.json
{
  "hooks": {
    "Stop": []
  }
}

# 或强制停止
/cancel-ralph
```

### 问题：迭代无进展

**原因**: 任务定义不清或方向错误

**解决方案**:
```bash
# 停止当前循环
/cancel-ralph

# 重新定义任务
/ralph-loop 更具体的任务描述 --iterations 5
```

### 问题：超时

**原因**: 任务过大或迭代次数过多

**解决方案**:
```bash
# 增加超时时间
/ralph-loop 任务 --timeout 120

# 或减少迭代次数
/ralph-loop 任务 --iterations 5
```
