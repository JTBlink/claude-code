### agent-sdk-dev

**功能**: Claude Agent SDK 开发工具包

**命令**:
- `/new-sdk-app` - 交互式创建新的 Agent SDK 项目

**Agent**:
- `agent-sdk-verifier-py` - 验证 Python SDK 应用
- `agent-sdk-verifier-ts` - 验证 TypeScript SDK 应用

**使用方式**:
```bash
# 创建新项目
/new-sdk-app

# 验证 Python 项目
@agent-sdk-verifier-py 验证这个项目

# 验证 TypeScript 项目
@agent-sdk-verifier-ts 检查代码规范
```

**详细文档**: [agent-sdk-dev/README.md](./agent-sdk-dev/README.md)

### code-review

**功能**: 自动化 PR 代码审查，使用多个 specialized agents

**命令**:
- `/code-review [选项]`

**审查 Agent** (5 个并行):
1. **CLAUDE.md 合规检查** - 检查规范遵循
2. **Bug 扫描** - 检测明显错误
3. **历史上下文** - 分析 Git 历史
4. **PR 历史** - 查看之前的评论
5. **代码注释** - 检查注释质量

**选项**:
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--comment` | 在 PR 中添加评论 | false |
| `--confidence <阈值>` | 置信度阈值 | 80 |

**使用方式**:
```bash
# 标准审查
/code-review

# 添加评论到 PR
/code-review --comment

# 调整置信度
/code-review --confidence 90
```

**详细文档**: [code-review/README.md](./code-review/README.md)

### feature-dev

**功能**: 7 阶段结构化功能开发工作流

**命令**:
- `/feature-dev <功能描述>`

**阶段**:
1. **发现** - 澄清需求，理解要构建什么
2. **探索** - 分析现有代码库和模式
3. **提问** - 解决所有模糊点
4. **设计** - 提供多个架构方案
5. **实现** - 编写代码
6. **审查** - 质量检查和修复
7. **总结** - 记录完成内容

**Agent**:
- `code-explorer` - 代码库分析
- `code-architect` - 架构设计
- `code-reviewer` - 质量审查

**使用方式**:
```bash
/feature-dev 添加 OAuth 用户认证功能
```

**详细文档**: [feature-dev/README.md](./feature-dev/README.md)

### hookify

**功能**: 简易创建自定义 Hooks，防止不良行为

**命令**:
| 命令 | 说明 |
|------|------|
| `/hookify <规则>` | 创建新规则 |
| `/hookify:list` | 列出所有规则 |
| `/hookify:configure` | 配置规则 |
| `/hookify:help` | 显示帮助 |

**Agent**:
- `conversation-analyzer` - 分析对话识别问题行为

**Skill**:
- `writing-rules` - Rule 语法指导

**使用方式**:
```bash
# 创建警告规则
/hookify 警告我使用 rm -rf 命令

# 创建阻止规则
/hookify 阻止我使用 console.log

# 列出规则
/hookify:list
```

**规则示例**:
```markdown
---
name: block-dangerous-rm
enabled: true
event: bash
pattern: rm\s+-rf
action: block
---

⚠️ **危险命令！** 此命令可能删除重要文件。
```

**详细文档**: [hookify/README.md](./hookify/README.md)

**命令**:

**审查方面**:

|------|-------|------|

| `types` | `type-design-analyzer` | 类型设计审查 |


### security-guidance

**功能**: 安全提醒钩子，监控代码编辑时的安全问题

**Hook**: `PreToolUse` - 监控文件写入操作

**监控模式** (9 种):
1. **命令注入** - 检测不安全的命令拼接
2. **XSS 漏洞** - 检测不安全的 HTML 输出
3. **eval 使用** - 检测 eval() 调用
4. **危险 HTML** - 检测 innerHTML 赋值
5. **Pickle 反序列化** - 检测不安全的 pickle 使用
6. **os.system 调用** - 检测系统命令执行
7. **SQL 注入** - 检测不安全的 SQL 拼接
8. **路径遍历** - 检测不安全的路径拼接
9. **敏感数据** - 检测敏感信息硬编码

**配置**:
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "file",
            "file": "plugins/security-guidance/hooks/security-check.py"
          }
        ]
      }
    ]
  }
}
```

**详细文档**: [security-guidance/README.md](./security-guidance/README.md)

## 插件安装

### 从市场安装

```bash
# 列出可用插件
/plugin list

# 安装插件
/plugin install <插件名>
```

### 本地开发测试

```bash
# 指定插件目录
claude --plugin-dir /path/to/plugins
```

### 项目配置

在 `.claude/settings.json` 中配置：

```json
{
  "plugins": {
    "enabled": ["feature-dev", "code-review", "hookify"]
  }
}
```
