# security-guidance 插件

> 代码安全提醒钩子，在编辑文件时警告潜在的安全问题

## 功能说明

`security-guidance` 插件通过 Hook 系统监控文件写入操作，检测并警告以下 9 种安全模式：

| # | 安全模式 | 检测内容 | 风险等级 |
|---|----------|----------|----------|
| 1 | **命令注入** | 不安全的命令拼接 | 🔴 高 |
| 2 | **XSS 漏洞** | 不安全的 HTML 输出 | 🔴 高 |
| 3 | **eval 使用** | eval() 调用 | 🔴 高 |
| 4 | **危险 HTML** | innerHTML 赋值 | 🟠 中 |
| 5 | **Pickle 反序列化** | 不安全的 pickle 使用 | 🔴 高 |
| 6 | **os.system 调用** | 系统命令执行 | 🟠 中 |
| 7 | **SQL 注入** | 不安全的 SQL 拼接 | 🔴 高 |
| 8 | **路径遍历** | 不安全的路径拼接 | 🟠 中 |
| 9 | **敏感数据** | 敏感信息硬编码 | 🟡 低 |

## 安装方式

```bash
# 从插件目录安装
claude --plugin-dir /path/to/claude-code/plugins

# 或在项目中启用
# 编辑 .claude/settings.json
{
  "plugins": {
    "enabled": ["security-guidance"]
  }
}
```

## 配置方式

### 基本配置

在 `.claude/settings.json` 中添加：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "file",
            "file": "/path/to/security-guidance/hooks/security-check.py"
          }
        ]
      }
    ]
  }
}
```

### Hook 配置

**位置**: `hooks/hooks.json`

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "python3 hooks/security-check.py"
          }
        ]
      }
    ]
  }
}
```

## 检测规则详解

### 1. 命令注入

**检测模式**:
```python
r"os\.system\s*\(\s*['\"].*%\s*"  # 字符串格式化
r"subprocess\..*shell\s*=\s*True"  # shell=True
r"exec\s*\(\s*user"  # exec 用户输入
```

**示例代码**:
```python
# ❌ 不安全
os.system("ping " + user_input)
exec(user_code)

# ✅ 安全
subprocess.run(["ping", host], check=True)
```

**警告消息**:
```
⚠️ **命令注入风险！**

检测到不安全的命令执行，可能导致命令注入攻击。
建议使用 subprocess 模块并避免 shell=True。
```

---

### 2. XSS 漏洞

**检测模式**:
```python
r"return\s+.*\+.*user"  # 字符串拼接返回
r"render.*user_input"  # 渲染用户输入
```

**示例代码**:
```python
# ❌ 不安全
return "<h1>" + username + "</h1>"

# ✅ 安全
return escape(username)
```

**警告消息**:
```
⚠️ **XSS 风险！**

检测到可能的跨站脚本漏洞。
请对用户输入进行转义后再输出。
```

---

### 3. eval 使用

**检测模式**:
```python
r"\beval\s*\("
r"\bexec\s*\("
```

**示例代码**:
```python
# ❌ 不安全
result = eval(user_expression)

# ✅ 安全
import ast
result = ast.literal_eval(user_expression)
```

**警告消息**:
```
⚠️ **eval 使用风险！**

eval() 和 exec() 可能执行任意代码。
建议使用 ast.literal_eval() 或其他安全替代方案。
```

---

### 4. 危险 HTML

**检测模式**:
```python
r"\.innerHTML\s*="
r"\.outerHTML\s*="
```

**示例代码**:
```python
# ❌ 不安全
element.innerHTML = userInput

# ✅ 安全
element.textContent = userInput
```

**警告消息**:
```
⚠️ **危险 HTML 操作！**

innerHTML 赋值可能导致 XSS 攻击。
建议使用 textContent 或 DOM 方法。
```

---

### 5. Pickle 反序列化

**检测模式**:
```python
r"pickle\.load\s*\("
r"pickle\.loads\s*\("
```

**示例代码**:
```python
# ❌ 不安全
data = pickle.load(untrusted_file)

# ✅ 安全
import json
data = json.load(trusted_file)
```

**警告消息**:
```
⚠️ **Pickle 反序列化风险！**

pickle.load() 可能执行恶意代码。
建议使用 JSON 或其他安全格式。
```

---

### 6. os.system 调用

**检测模式**:
```python
r"os\.system\s*\("
r"os\.popen\s*\("
```

**示例代码**:
```python
# ❌ 不安全
os.system("ls " + directory)

# ✅ 安全
subprocess.run(["ls", directory], capture_output=True)
```

**警告消息**:
```
⚠️ **os.system 调用！**

os.system() 存在安全风险。
建议使用 subprocess 模块替代。
```

---

### 7. SQL 注入

**检测模式**:
```python
r"execute\s*\(\s*['\"].*%\s*"  # 字符串格式化
r"query\s*\(\s*.*\+.*"  # 字符串拼接
```

**示例代码**:
```python
# ❌ 不安全
cursor.execute("SELECT * FROM users WHERE id = " + user_id)

# ✅ 安全
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))
```

**警告消息**:
```
⚠️ **SQL 注入风险！**

检测到不安全的 SQL 拼接。
请使用参数化查询防止 SQL 注入。
```

---

### 8. 路径遍历

**检测模式**:
```python
r"open\s*\(\s*.*\+.*"  # 路径拼接
r"path\.join\s*\(.*user"  # 用户输入路径
```

**示例代码**:
```python
# ❌ 不安全
open("/data/" + filename)

# ✅ 安全
import os
safe_path = os.path.join(BASE_DIR, filename)
open(safe_path)
```

**警告消息**:
```
⚠️ **路径遍历风险！**

检测到不安全的路径拼接。
请验证路径在预期目录内。
```

---

### 9. 敏感数据

**检测模式**:
```python
r"password\s*=\s*['\"][^'\"]+['\"]"  # 硬编码密码
r"api_key\s*=\s*['\"][^'\"]+['\"]"  # 硬编码 API 密钥
r"secret\s*=\s*['\"][^'\"]+['\"]"  # 硬编码密钥
```

**示例代码**:
```python
# ❌ 不安全
DB_PASSWORD = "supersecret123"
API_KEY = "sk-abc123..."

# ✅ 安全
import os
DB_PASSWORD = os.environ.get("DB_PASSWORD")
API_KEY = os.environ.get("API_KEY")
```

**警告消息**:
```
⚠️ **敏感数据硬编码！**

检测到硬编码的敏感信息。
请使用环境变量或密钥管理服务。
```

## 使用示例

### 运行安全检查

```bash
# 在会话中自动触发
# 当写入文件时，Hook 会自动检查

# 或手动运行检查脚本
python3 hooks/security-check.py < input.json
```

### 输入格式

Hook 脚本接收 JSON 输入：

```json
{
  "tool_input": {
    "file_path": "src/auth.py",
    "content": "password = 'secret123'\n..."
  }
}
```

### 输出格式

```json
{
  "allowed": false,
  "reason": "检测到敏感数据硬编码",
  "details": [
    {
      "line": 1,
      "pattern": "password",
      "suggestion": "使用环境变量存储密码"
    }
  ]
}
```

## 自定义规则

### 添加新规则

编辑 `hooks/security-check.py`:

```python
_CUSTOM_RULES = [
    (
        r"custom_pattern",
        "自定义警告消息",
        "建议的修复方案"
    ),
]
```

### 禁用规则

在配置中指定禁用的规则：

```json
{
  "securityGuidance": {
    "disabledRules": ["pickle", "eval"]
  }
}
```

### 调整风险等级

```json
{
  "securityGuidance": {
    "riskLevels": {
      "command-injection": "critical",
      "xss": "high",
      "hardcoded-secrets": "medium"
    }
  }
}
```

## 最佳实践

### 安全编码指南

1. **输入验证**: 始终验证和清理用户输入
2. **参数化查询**: 使用参数化防止 SQL 注入
3. **转义输出**: 对所有输出进行适当转义
4. **最小权限**: 使用最小必要权限运行
5. **密钥管理**: 使用环境变量或密钥管理服务

### 修复建议

| 问题 | 修复方案 |
|------|----------|
| 命令注入 | 使用 subprocess，避免 shell=True |
| XSS | 使用模板引擎自动转义 |
| eval | 使用 ast.literal_eval |
| SQL 注入 | 使用 ORM 或参数化查询 |
| 路径遍历 | 验证路径在白名单内 |

## 故障排查

### 问题：Hook 未触发

**诊断**:
```bash
# 检查 Hook 配置
cat .claude/settings.json | jq .hooks

# 测试脚本
echo '{"tool_input": {"content": "eval(x)"}}' | python3 hooks/security-check.py
```

### 问题：误报过多

**解决方案**:
```json
{
  "securityGuidance": {
    "ignorePatterns": [
      "test_.*\\.py",  // 忽略测试文件
      ".*\\.test\\.js"
    ]
  }
}
```

### 问题：性能影响

**解决方案**:
- 仅在关键文件上启用
- 使用缓存机制
- 异步执行检查
