## 快速入门

### 第一步：安装 Claude Code

#### macOS / Linux（推荐）

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

#### Homebrew（macOS / Linux）

```bash
brew install --cask claude-code
```

#### Windows（推荐）

```powershell
irm https://claude.ai/install.ps1 | iex
```

#### WinGet（Windows）

```powershell
winget install Anthropic.ClaudeCode
```

### 第二步：首次启动

```bash
# 导航到项目目录
cd /path/to/your/project

# 启动 Claude Code
claude
```

### 第三步：开始对话

```
> 帮我分析这个项目的结构

> 创建一个用户认证功能

> 修复这个 bug
```

### 第四步：尝试插件命令

```bash
# 功能开发
/feature-dev 添加用户登录功能

# 代码审查
/code-review

# Git 工作流
/commit-push-pr
```

## 最佳实践

### 与 Claude Code 高效沟通

#### ✅ 好的提示词

```
帮我重构这个函数，使其更符合 TypeScript 最佳实践：
- 添加类型注解
- 使用 async/await 替代 Promise.then
- 添加错误处理
- 函数体控制在 30 行以内
```

#### ❌ 避免模糊的提示词

```
改进这个代码
```

### 任务分解技巧

将大任务拆解为小步骤：

```
1. 首先，分析现有的用户认证流程
2. 然后，设计新的 OAuth 集成方案
3. 接着，实现 GitHub OAuth 登录
4. 最后，添加单元测试
```

### 上下文管理

- **小项目**: 直接让 Claude 分析整个项目
- **大项目**: 指定关键文件目录
- **复杂任务**: 使用 `/compact` 压缩历史

### Git 工作流自动化

使用内置命令简化 Git 操作：

```bash
# 一键提交、推送、创建 PR
/commit-push-pr

# Issue 自动分类
/triage-issue

# Issue 去重
/dedupe
```

### 代码审查最佳实践

1. **运行审查命令**:
   ```bash
   /code-review --confidence 80
   ```

2. **审查多个方面**:
   - CLAUDE.md 规范合规性
   - 明显 Bug 扫描
   - 历史上下文分析
   - 测试覆盖检查

3. **过滤误报**: 置信度低于阈值的反馈可忽略

### 安全配置建议

#### 个人开发（宽松配置）

```json
{
  "permissions": {
    "disableBypassPermissionsMode": "disable"
  }
}
```

#### 企业环境（严格配置）

```json
{
  "permissions": {
    "disableBypassPermissionsMode": "disable",
    "ask": ["Bash"],
    "deny": ["WebSearch", "WebFetch"]
  },
  "allowManagedPermissionRulesOnly": true,
  "allowManagedHooksOnly": true
}
```

#### Bash 沙箱（最高安全）

```json
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": false,
    "allowUnsandboxedCommands": false
  }
}
```

## 故障排查

### 常见问题

#### Q1: Claude Code 无法启动

**症状**: 运行 `claude` 命令无响应

**解决方案**:
```bash
# 检查 Node.js 版本
node --version  # 需要 18+

# 重新安装
curl -fsSL https://claude.ai/install.sh | bash

# 清除缓存
rm -rf ~/.claude/cache
```

#### Q2: 插件不工作

**症状**: `/plugin list` 看不到已安装的插件

**解决方案**:
```bash
# 检查插件目录
ls -la .claude/

# 验证 plugin.json
cat .claude-plugin/plugin.json

# 重启 Claude Code
exit
claude
```

#### Q3: Git 命令失败

**症状**: `/commit-push-pr` 执行失败

**解决方案**:
```bash
# 检查 Git 状态
git status

# 检查 GitHub CLI
gh --version

# 重新登录 GitHub
gh auth logout
gh auth login
```

#### Q4: Hook 不触发

**症状**: 自定义 Hook 没有生效

**解决方案**:
```bash
# 检查 Hook 配置
cat .claude/settings.json | jq .hooks

# 验证 Hook 文件语法
python3 -m json.tool .claude/hooks/my-hook.json

# 启用详细日志
claude --verbose
```

#### Q5: 模型切换失败

**症状**: `/model` 命令无效

**解决方案**:
```bash
# 检查可用模型
claude --help

# 使用命令行参数
claude --model claude-sonnet-4-5-20250929

# 检查配置文件
cat ~/.claude/settings.json | jq .model
```

### 调试技巧

#### 启用详细日志

```bash
claude --verbose
```

#### 查看日志文件

```bash
# 用户级日志
~/.claude/logs/

# 项目级日志
.claude/logs/
```

#### 测试 Hook 脚本

```bash
# 独立运行 Hook 脚本
python3 /path/to/hook.py < test-input.json

# 检查输出
echo $?  # 退出码：0=允许，2=阻止
```

### 性能优化

#### 减少 Token 使用

- 使用 `/compact` 压缩会话历史
- 避免一次性加载过多文件
- 定期清除不相关的上下文

#### 加快响应速度

- 简单任务使用 Haiku 模型
- 指定关键文件而非整个项目
- 任务分解为小步骤

#### 优化 Git 工作流

- 小提交优于大提交
- 使用 `/commit-push-pr` 自动化
- 定期同步远程分支
