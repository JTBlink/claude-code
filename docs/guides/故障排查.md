## 安装问题

### 问题：安装脚本失败

**症状**:
```bash
$ curl -fsSL https://claude.ai/install.sh | bash
curl: (6) Could not resolve host: claude.ai
```

**原因**: 网络连接问题

**解决方案**:
```bash
# 检查网络连接
ping claude.ai

# 使用备用安装方式
brew install --cask claude-code  # macOS/Linux
winget install Anthropic.ClaudeCode  # Windows

# 使用 NPM（已弃用，不推荐）
npm install -g @anthropic-ai/claude-code
```

### 问题：Node.js 版本过低

**症状**:
```
Error: Node.js 18 or higher is required.
Current version: v16.14.0
```

**解决方案**:
```bash
# 查看当前版本
node --version

# 使用 nvm 升级（推荐）
nvm install 18
nvm use 18

# 或直接下载最新版
# https://nodejs.org/
```

### 问题：权限不足

**症状**:
```bash
npm ERR! Error: EACCES: permission denied
```

**解决方案**:
```bash
# macOS/Linux - 修复 npm 权限
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 或使用 sudo（不推荐）
sudo npm install -g @anthropic-ai/claude-code
```

## 启动问题

### 问题：claude 命令找不到

**症状**:
```bash
$ claude
command not found: claude
```

**解决方案**:
```bash
# 检查安装位置
which claude

# 检查 PATH
echo $PATH

# 添加 PATH（macOS/Linux）
export PATH=~/.npm-global/bin:$PATH
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc

# Windows - 检查环境变量
# 系统属性 > 环境变量 > Path
```

### 问题：启动后无响应

**症状**: Claude Code 启动后卡住，无输入提示

**解决方案**:
```bash
# 清除缓存
rm -rf ~/.claude/cache

# 清除配置（备份后）
mv ~/.claude ~/.claude.backup

# 重新启动
claude

# 检查终端兼容性
# 推荐使用：iTerm2, Windows Terminal, alacritty
```

### 问题：认证失败

**症状**:
```
Authentication failed. Please check your API key.
```

**解决方案**:
```bash
# 重新登录
claude login

# 检查 API Key
# https://console.anthropic.com/

# 检查网络连接
# 某些地区需要代理
```

## 命令执行问题

### 问题：斜杠命令无效

**症状**:
```
Unknown command: /feature-dev
```

**原因**: 插件未安装

**解决方案**:
```bash
# 查看已安装插件
/plugin list

# 安装插件
/plugin install feature-dev

# 检查插件目录
ls -la .claude/

# 重启 Claude Code
exit
claude
```

### 问题：Bash 命令被阻止

**症状**:
```
⚠️ Bash command execution requires permission.
```

**解决方案**:
```bash
# 临时允许
# 在提示时选择 "Allow"

# 配置权限
# 编辑 .claude/settings.json
{
  "permissions": {
    "ask": []  # 移除 Bash
  }
}

# 或使用沙箱模式
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true
  }
}
```

### 问题：文件写入失败

**症状**:
```
Error: Unable to write file. Permission denied.
```

**解决方案**:
```bash
# 检查文件权限
ls -la path/to/file

# 修复权限
chmod u+w path/to/file

# 检查是否只读挂载
mount | grep project

# 使用 sudo 运行（不推荐）
sudo claude
```

## 插件问题

### 问题：插件不工作

**症状**: 插件命令无响应或报错

**诊断步骤**:
```bash
# 1. 检查插件是否安装
/plugin list

# 2. 检查插件目录结构
tree .claude/

# 3. 验证 plugin.json
cat .claude-plugin/plugin.json | jq .

# 4. 检查命令文件
cat .claude/commands/<command>.md
```

**常见问题**:

| 问题 | 解决方案 |
|------|----------|
| plugin.json 格式错误 | 使用 JSON 验证器检查 |
| 命令文件不存在 | 重新安装插件 |
| 插件版本不兼容 | 更新到最新版 |

### 问题：Hook 不触发

**症状**: 自定义 Hook 没有生效

**诊断步骤**:
```bash
# 1. 检查 Hook 配置
cat .claude/settings.json | jq .hooks

# 2. 验证 Hook 文件
cat .claude/hooks/my-hook.json

# 3. 检查事件匹配
# 确认事件类型正确：PreToolUse, PostToolUse, etc.

# 4. 启用详细日志
claude --verbose
```

**Hook 配置示例**:
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "python3 /path/to/validator.py"
          }
        ]
      }
    ]
  }
}
```

### 问题：插件冲突

**症状**: 多个插件同时生效导致异常

**解决方案**:
```bash
# 1. 列出所有插件
/plugin list

# 2. 禁用冲突插件
# 编辑 .claude/settings.json
{
  "disabledPlugins": ["plugin-name"]
}

# 3. 或使用互斥的插件
# 例如：只使用 code-review 或 pr-review-toolkit 之一
```

## Git 集成问题

### 问题：/commit-push-pr 失败

**症状**: 命令执行中途失败

**诊断步骤**:
```bash
# 1. 检查 Git 状态
git status

# 2. 检查远程仓库
git remote -v

# 3. 检查 GitHub CLI
gh --version
gh auth status

# 4. 检查分支
git branch -a
```

**常见问题**:

| 问题 | 解决方案 |
|------|----------|
| 未登录 GitHub | `gh auth login` |
| 远程仓库不存在 | `git remote add origin <url>` |
| 权限不足 | 检查仓库访问权限 |
| 分支已存在 | 删除或重命名分支 |

### 问题：GitHub CLI 未安装

**症状**:
```
gh: command not found
```

**解决方案**:
```bash
# macOS
brew install gh

# Linux
sudo apt install gh  # Debian/Ubuntu
sudo dnf install gh  # Fedora

# Windows
winget install GitHub.cli

# 验证安装
gh --version

# 登录
gh auth login
```

### 问题：PR 创建失败

**症状**:
```
Error: Failed to create pull request
```

**可能原因**:
1. 基础分支不存在
2. 权限不足
3. PR 已存在

**解决方案**:
```bash
# 1. 检查基础分支
git checkout main
git pull origin main

# 2. 检查权限
gh repo view

# 3. 手动创建 PR
gh pr create \
  --title "标题" \
  --body "描述" \
  --base main
```

## 性能问题

### 问题：响应速度慢

**症状**: Claude 响应时间超过 30 秒

**解决方案**:
```bash
# 1. 切换到更快的模型
/model claude-haiku-3-5

# 2. 减少上下文
/compact

# 3. 任务分解
# 将大任务拆分为小步骤

# 4. 检查网络连接
# 某些地区需要代理
```

### 问题：Token 使用过快

**症状**: 会话很快达到 token 限制

**解决方案**:
```bash
# 1. 压缩历史
/compact

# 2. 清除无关上下文
/clear

# 3. 避免加载大文件
# 使用 /read 只读关键部分

# 4. 开启新会话
exit
claude
```

### 问题：内存占用高

**症状**: 系统内存占用持续增长

**解决方案**:
```bash
# 1. 重启 Claude Code
exit

# 2. 清除缓存
rm -rf ~/.claude/cache

# 3. 减少并发操作
# 避免同时运行多个 Agent
```

## 调试技巧

### 启用详细日志

```bash
claude --verbose
```

### 查看日志文件

```bash
# 用户级日志
ls -la ~/.claude/logs/

# 项目级日志
ls -la .claude/logs/

# 实时查看日志
tail -f ~/.claude/logs/claude.log
```

### 测试 Hook 脚本

```bash
# 创建测试输入
echo '{"tool_input": {"command": "rm -rf /"}}' > test-input.json

# 运行 Hook 脚本
python3 /path/to/hook.py < test-input.json

# 检查退出码
echo $?  # 0=允许，2=阻止
```

### 网络问题诊断

```bash
# 检查 API 连接
curl -I https://api.anthropic.com

# 检查代理设置
echo $HTTP_PROXY
echo $HTTPS_PROXY

# 测试 DNS
nslookup claude.ai
```

### 使用开发者工具

```bash
# 检查配置
claude --config

# 检查插件
claude --plugin-dir /path/to/plugin

# 恢复会话
claude --resume
```
