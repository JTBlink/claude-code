## 设置配置

### 配置文件层次

```
系统默认 < 用户全局 < 项目级 < 工作树级 < 命令行参数
```

### 文件位置

| 级别 | 位置 | 说明 |
|------|------|------|
| 全局 | `~/.claude/settings.json` | 用户级配置 |
| 项目 | `.claude/settings.json` | 项目级配置 |
| 本地 | `.claude/settings.local.json` | 本地覆盖（不提交） |
| 管理 | `managed-settings.json` | 企业强制策略 |

---

### 宽松配置 (settings-lax.json)

**适用场景**: 个人开发，限制较少

**文件**: [settings-lax.json](./settings/settings-lax.json)

```json
{
  "permissions": {
    "disableBypassPermissionsMode": "disable"
  },
  "strictKnownMarketplaces": []
}
```

**说明**:
- 允许绕过权限模式
- 不限制市场来源

---

### 严格配置 (settings-strict.json)

**适用场景**: 企业环境，严格管控

**文件**: [settings-strict.json](./settings/settings-strict.json)

```json
{
  "permissions": {
    "disableBypassPermissionsMode": "disable",
    "ask": ["Bash"],
    "deny": ["WebSearch", "WebFetch"]
  },
  "allowManagedPermissionRulesOnly": true,
  "allowManagedHooksOnly": true,
  "sandbox": {
    "autoAllowBashIfSandboxed": false
  }
}
```

**说明**:
- Bash 命令需要确认
- 禁止网络搜索和抓取
- 仅允许管理的权限规则
- 仅允许管理的 Hooks

---

### Bash 沙箱配置 (settings-bash-sandbox.json)

**适用场景**: 最高安全要求

**文件**: [settings-bash-sandbox.json](./settings/settings-bash-sandbox.json)

```json
{
  "allowManagedPermissionRulesOnly": true,
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": false,
    "allowUnsandboxedCommands": false
  }
}
```

**说明**:
- 强制沙箱模式
- 所有 Bash 命令在沙箱中运行
- 不允许非沙箱命令

## Hooks 配置

### Hook 文件结构

```json
{
  "hooks": {
    "<事件类型>": [
      {
        "matcher": "<匹配器>",
        "hooks": [
          {
            "type": "command",
            "command": "<执行命令>"
          }
        ]
      }
    ]
  }
}
```

### 支持的事件类型

| 事件 | 触发时机 | 用途 |
|------|----------|------|
| `PreToolUse` | 工具调用前 | 验证、修改工具输入 |
| `PostToolUse` | 工具调用后 | 处理工具输出 |
| `SessionStart` | 会话开始时 | 注入上下文、设置 |
| `SessionEnd` | 会话结束时 | 清理、总结 |
| `Stop` | Agent 想要停止时 | 检查完成条件 |
| `Prompt` | 用户提交提示时 | 预处理用户输入 |

### Bash 命令验证器示例

**文件**: [bash_command_validator_example.py](./hooks/bash_command_validator_example.py)

**功能**: 验证 Bash 命令，阻止危险操作

```python
#!/usr/bin/env python3
import json
import re
import sys

_VALIDATION_RULES = [
    (r"^grep\b(?!.*\|)", "使用 'rg' 代替 'grep' 以获得更好性能"),
    (r"^find\s+\S+\s+-name\b", "使用 'rg --files' 代替 'find -name'"),
]

def main():
    input_data = json.load(sys.stdin)
    command = input_data.get("tool_input", {}).get("command", "")

    for pattern, message in _VALIDATION_RULES:
        if re.search(pattern, command):
            print(f"• {message}", file=sys.stderr)
            sys.exit(2)  # 阻止工具调用

if __name__ == "__main__":
    main()
```

**配置方式**:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "python3 /path/to/validator.py"
          }
        ]
      }
    ]
  }
}
```

### Hookify 规则示例

使用 [hookify](../../plugins/hookify/Hookify 插件.md) 插件创建规则：

```bash
# 创建警告规则
/hookify 警告我使用 rm -rf 命令

# 创建阻止规则
/hookify 阻止我提交 .env 文件
```

**生成的规则文件** (`.claude/hookify.warn-rm.local.md`):

```markdown
---
name: block-dangerous-rm
enabled: true
event: bash
pattern: rm\s+-rf
action: block
---

⚠️ **危险命令检测！**

此命令可能删除重要文件，请确认路径正确。
```

## 权限配置

### 权限级别

| 级别 | 说明 |
|------|------|
| `allow` | 允许，无需确认 |
| `ask` | 询问，需要确认 |
| `deny` | 拒绝，禁止使用 |

### 工具权限配置

```json
{
  "permissions": {
    "allow": ["Read", "Glob"],
    "ask": ["Bash", "Write"],
    "deny": ["WebSearch", "WebFetch"]
  }
}
```

### Bash 权限配置

```json
{
  "permissions": {
    "ask": ["Bash"]
  },
  "sandbox": {
    "enabled": true,
    "allowedCommands": ["ls", "cat", "grep", "rg"],
    "deniedCommands": ["rm -rf", "sudo", "curl"]
  }
}
```

### 网络权限配置

```json
{
  "permissions": {
    "ask": ["WebFetch"]
  },
  "network": {
    "allowLocalBinding": false,
    "allowedDomains": ["api.example.com", "github.com"],
    "deniedDomains": ["malicious.com"]
  }
}
```

## 使用指南

### 创建项目配置

```bash
# 创建 .claude 目录
mkdir -p .claude

# 创建配置文件
cat > .claude/settings.json << 'EOF'
{
  "model": "claude-sonnet-4-5-20250929",
  "permissions": {
    "ask": ["Bash"]
  }
}
EOF
```

### 创建本地覆盖

```bash
# 创建不提交的本地配置
cat > .claude/settings.local.json << 'EOF'
{
  "model": "claude-opus-4-6-20250929"
}
EOF

# 添加到 .gitignore
echo ".claude/settings.local.json" >> .gitignore
```

### 验证配置

```bash
# 查看当前配置
claude --config

# 或在会话中查看
/model
```

## 最佳实践

### 团队配置共享

```bash
# 提交项目配置
git add .claude/settings.json
git commit -m "chore: 添加 Claude Code 配置"

# 个人配置不提交
echo ".claude/settings.local.json" >> .gitignore
echo ".claude/settings.*.local.json" >> .gitignore
```

### 环境特定配置

```bash
# 开发环境
cat > .claude/settings.dev.json << 'EOF'
{
  "model": "claude-haiku-3-5",
  "sandbox": { "enabled": false }
}
EOF

# 生产环境
cat > .claude/settings.prod.json << 'EOF'
{
  "model": "claude-sonnet-4-5-20250929",
  "sandbox": { "enabled": true }
}
EOF
```

### 配置继承

```
~/.claude/settings.json          # 全局默认
    ↓
.claude/settings.json            # 项目级
    ↓
.claude/settings.local.json      # 本地覆盖
    ↓
命令行参数                       # 最高优先级
```
