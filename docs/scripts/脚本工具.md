## GitHub CLI 封装

### gh.sh

**功能**: GitHub CLI 的受限封装，提供安全的 GitHub 操作

**位置**: `scripts/gh.sh`

**支持的命令**:

```bash
# 查看 Issue
./scripts/gh.sh issue view <编号>

# 列出 Issue
./scripts/gh.sh issue list --state open

# 搜索 Issue
./scripts/gh.sh search issues "<关键词>" --limit 10

# 列出标签
./scripts/gh.sh label list --limit 100

# 添加标签
./scripts/gh.sh label add <Issue 编号> <标签>

# 关闭 Issue
./scripts/gh.sh issue close <编号>

# 创建 Issue
./scripts/gh.sh issue create --title "标题" --body "描述"
```

**安全限制**:
- 仅限只读操作（部分）
- 限制批量操作数量
- 记录所有操作日志

**使用示例**:

```bash
# 查看 Issue #42 的详情
./scripts/gh.sh issue view 42

# 列出所有打开的 Bug
./scripts/gh.sh issue list --state open --label bug

# 搜索包含"登录"的 Issue
./scripts/gh.sh search issues "登录" --limit 20

# 查看前 50 个标签
./scripts/gh.sh label list --limit 50
```

## Issue 管理脚本

### sweep.ts

**功能**: Issue 自动管理，标记过期和自动关闭

**位置**: `scripts/sweep.ts`

**运行方式**:
```bash
npx tsx scripts/sweep.ts
```

**功能**:
1. **标记过期**: 长时间无活动的 Issue 标记为 `stale`
2. **自动关闭**: 过期 Issue 自动关闭
3. **热度保护**: 高热度 Issue 不自动关闭

**配置选项**:
```bash
# 设置过期天数
npx tsx scripts/sweep.ts --stale-days 30

# 设置关闭天数
npx tsx scripts/sweep.ts --close-days 90

# 模拟运行（不实际执行）
npx tsx scripts/sweep.ts --dry-run
```

**详细文档**: [sweep.ts.md](./sweep.ts.md)

## 工作流脚本

### edit-issue-labels.sh

**功能**: 批量编辑 Issue 标签

**位置**: `scripts/edit-issue-labels.sh`

**运行方式**:
```bash
./scripts/edit-issue-labels.sh <Issue 编号> <操作> <标签>
```

**操作**:
- `add` - 添加标签
- `remove` - 移除标签
- `set` - 设置标签（替换所有）

**示例**:
```bash
# 添加 bug 标签
./scripts/edit-issue-labels.sh 42 add bug

# 移除 stale 标签
./scripts/edit-issue-labels.sh 42 remove stale

# 设置标签
./scripts/edit-issue-labels.sh 42 set "bug,priority:high"
```

## 使用指南

### 前置条件

```bash
# 安装 Node.js 18+
node --version

# 安装 tsx（TypeScript 执行器）
npm install -g tsx

# 安装 GitHub CLI
brew install gh  # macOS
sudo apt install gh  # Linux

# 登录 GitHub
gh auth login
```

### 权限要求

```bash
# 检查仓库权限
gh repo view

# 需要以下权限：
# - issues:read - 读取 Issue
# - issues:write - 编辑 Issue
# - labels:read - 读取标签
# - labels:write - 编辑标签
```

### 运行脚本

```bash
# TypeScript 脚本
npx tsx scripts/<脚本名>.ts [参数]

# Shell 脚本
chmod +x scripts/<脚本名>.sh
./scripts/<脚本名>.sh [参数]
```

### 模拟运行

建议先模拟运行，确认无误后再实际执行：

```bash
# 大多数脚本支持 --dry-run 参数
npx tsx scripts/sweep.ts --dry-run
npx tsx scripts/auto-close-duplicates.ts --dry-run
```

### 日志查看

```bash
# 脚本日志输出到控制台
# GitHub Actions 日志在 Actions 标签页查看
```

## GitHub Actions 集成

这些脚本通常与 GitHub Actions 配合使用：

### 工作流示例

```yaml
# .github/workflows/issue-lifecycle.yml
name: Issue Lifecycle Management

on:
  issues:
    types: [opened, closed, labeled]
  schedule:
    - cron: '0 0 * * *'  # 每天运行

jobs:
  lifecycle:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm install tsx
      
      - name: Run lifecycle script
        run: npx tsx scripts/issue-lifecycle.ts
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 故障排查

### 问题：脚本执行失败

**诊断步骤**:
```bash
# 检查 Node.js 版本
node --version  # 需要 18+

# 检查 tsx 安装
npx tsx --version

# 检查 GitHub CLI
gh --version
gh auth status

# 检查权限
gh repo view
```

### 问题：权限不足

**解决方案**:
```bash
# 检查个人访问令牌权限
# 需要：repo, workflow 权限

# 重新生成令牌
# https://github.com/settings/tokens

# 更新本地令牌
gh auth logout
gh auth login
```

### 问题：速率限制

**症状**:
```
API rate limit exceeded
```

**解决方案**:
```bash
# 等待重置（通常 1 小时）
# 或使用个人访问令牌提高限制
# https://github.com/settings/tokens
```
