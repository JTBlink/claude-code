## 项目自定义命令

本项目在 `.claude/commands/` 目录下定义的自定义命令。

### `/commit-push-pr`

**功能**: 一键完成代码提交、推送到远程、创建 Pull Request

**使用方式**:
```bash
/commit-push-pr
```

**执行流程**:
1. 检查当前分支，如果在 main 上则创建新分支
2. 暂存所有更改
3. 生成规范的提交信息（遵循 Conventional Commits）
4. 提交代码
5. 推送到远程仓库
6. 创建 Pull Request

**详细说明**: [提交推送 PR.md](./custom/提交推送%20PR.md)

---

### `/dedupe`

**功能**: Issue 去重，查找并合并重复的 Issue

**使用方式**:
```bash
/dedupe
```

**执行流程**:
1. 分析当前 Issue 的标题和内容
2. 搜索仓库中相似的 Issue
3. 识别重复的 Issue
4. 将重复 Issue 的评论迁移到主 Issue
5. 关闭重复 Issue 并添加引用

**详细说明**: [Issue 去重.md](./custom/Issue%20去重.md)

---

### `/triage-issue`

**功能**: Issue 自动分类和标记

**使用方式**:
```bash
/triage-issue
```

**执行流程**:
1. 分析 Issue 内容
2. 判断问题类型（Bug、功能请求、文档问题等）
3. 添加相应的标签
4. 分配给合适的维护者
5. 添加初步回复

**详细说明**: [Issue 分类.md](./custom/Issue%20分类.md)

## 插件命令

安装插件后可用的扩展命令。

### Feature Dev (`/feature-dev`)

**插件**: [feature-dev](../plugins/feature-dev/功能开发插件.md)

**命令**: `/feature-dev <功能描述>`

**说明**: 7 阶段结构化功能开发工作流

**阶段**:
1. 发现 - 澄清需求
2. 探索 - 分析代码库
3. 提问 - 解决模糊点
4. 设计 - 提供架构方案
5. 实现 - 编写代码
6. 审查 - 质量检查
7. 总结 - 记录完成内容

---

### Code Review (`/code-review`)

**插件**: [code-review](../plugins/code-review/代码审查插件.md)

**命令**: `/code-review [选项]`

**说明**: 自动化 PR 代码审查

**选项**:
- `--comment` - 在 PR 中添加评论
- `--confidence <阈值>` - 置信度阈值（默认 80）

**审查 Agent**:
- CLAUDE.md 合规性检查
- Bug 扫描
- 历史上下文分析
- PR 历史分析
- 代码注释检查

---

### Hookify (`/hookify`)

**插件**: [hookify](../plugins/hookify/Hookify%20插件.md)

**命令**:
| 命令 | 说明 |
|------|------|
| `/hookify <规则>` | 创建新的 Hook 规则 |
| `/hookify:list` | 列出所有规则 |
| `/hookify:configure` | 配置规则 |
| `/hookify:help` | 显示帮助 |

**示例**:
```bash
# 创建警告规则
/hookify 警告我使用 rm -rf 命令

# 创建阻止规则
/hookify 阻止我提交 console.log
```

### Plugin Dev (`/plugin-dev:create-plugin`)

**插件**: [plugin-dev](../plugins/plugin-dev/插件开发工具包.md)

**命令**: `/plugin-dev:create-plugin`

**说明**: 8 阶段引导式插件创建流程

**阶段**:
1. 插件信息收集
2. 目录结构创建
3. 插件元数据配置
4. 命令开发（可选）
5. Agent 开发（可选）
6. Skill 开发（可选）
7. Hook 开发（可选）
8. 验证与文档

---

### Ralph Loop (`/ralph-loop`)

**插件**: [ralph-wiggum](../plugins/ralph-wiggum/自主迭代循环插件.md)

**命令**:
| 命令 | 说明 |
|------|------|
| `/ralph-loop <任务>` | 启动自主迭代循环 |
| `/cancel-ralph` | 停止循环 |

**说明**: Claude 重复执行同一任务直到完成，适用于需要多次迭代的复杂任务。

### PR Review Toolkit (`/pr-review-toolkit:review-pr`)

**插件**: [pr-review-toolkit](../plugins/pr-review-toolkit/PR%20审查工具插件.md)

**命令**: `/pr-review-toolkit:review-pr [方面]`

**审查方面**:
| 方面 | 说明 |
|------|------|
| `comments` | 评论分析 |
| `tests` | 测试覆盖分析 |
| `errors` | 错误处理检查 |
| `types` | 类型设计审查 |
| `code` | 代码质量审查 |
| `simplify` | 代码简化建议 |
| `all` | 全部审查 |
