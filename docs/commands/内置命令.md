### `/btw` - 附带问题命令

**功能**: 在主任务进行时提问附带问题，不将对话添加到主会话历史，节省 token 消耗。

**使用方式**:
```bash
/btw [你的问题或上下文]
```

**核心优势**:
- **节省 Token**: 附带问题不会加入主对话历史，可减少高达 50% 的 token 消耗
- **不中断主任务**: Claude 工作时可随时提问
- **保持上下文整洁**: 主线程专注于核心任务

**适用场景**:
- 快速事实查询（"Python 的 asyncio.gather() 如何处理异常？"）
- 添加遗漏的上下文（"我们的数据库是 PostgreSQL 14"）
- 确认理解（"PUT 是完全替换，PATCH 是部分更新，对吧？"）
- 代码风格偏好（"请保持代码注释最小化"）

**示例**:
```bash
# 技术问题查询
/btw Redis 集群模式下 session 存储有什么限制？

# 添加约束条件
/btw 我们有 10MB 的 bundle 大小预算

# 确认概念
/btw Docker volume 和 bind mount 的区别是什么？

# 批量注入上下文
/btw 本项目的约定：(1) 使用 Black 格式化 (2) 需要类型提示 (3) PostgreSQL 14
```

**与其他命令对比**:

| 命令 | 作用 |
|------|------|
| `/btw` | 预防性 - 阻止附带对话进入主线程 |
| `/compact` | 纠正性 - 压缩已存在的历史 |
| `/clear` | 完全清除 - 开始全新会话 |

**注意事项**:
- `/btw` 上下文仅在当前会话有效，不会持久化
- 重要决策应使用普通消息，而非 `/btw`
- 需要后续参考的对话应放在主线程

## 命令行参数

运行 Claude Code 时可用的命令行参数。默认启动交互式会话，使用 `-p/--print` 进行非交互式输出。

### 基本用法

```bash
claude [options] [command] [prompt]
```

### 参数说明

#### 会话管理

| 参数 | 说明 |
|------|------|
| `-c, --continue` | 继续当前目录中最近的会话 |
| `-r, --resume [value]` | 通过会话 ID 恢复会话，或打开交互选择器 |
| `--fork-session` | 恢复时创建新会话 ID（与 --resume 或 --continue 配合） |
| `--session-id <uuid>` | 使用指定的会话 ID |
| `-n, --name <name>` | 设置会话显示名称（显示在 /resume 和终端标题） |
| `--no-session-persistence` | 禁用会话持久化（仅 --print 模式） |

#### 模型与 Agent

| 参数 | 说明 |
|------|------|
| `--model <model>` | 指定模型（别名如 'sonnet'、'opus'，或完整名称） |
| `--effort <level>` | Effort 级别（low, medium, high, max） |
| `--agent <agent>` | 当前会话使用的 Agent，覆盖 'agent' 设置 |
| `--agents <json>` | JSON 对象定义自定义 Agent |
| `--fallback-model <model>` | 默认模型过载时自动回退到指定模型（仅 --print） |

#### 权限与安全

| 参数 | 说明 |
|------|------|
| `--permission-mode <mode>` | 权限模式：default, acceptEdits, bypassPermissions, dontAsk, plan, auto |
| `--dangerously-skip-permissions` | 跳过所有权限检查（仅限无网络沙箱环境） |
| `--allow-dangerously-skip-permissions` | 启用跳过权限检查选项，但不默认启用 |
| `--allowedTools <tools...>` | 允许的工具列表（如 "Bash(git:*) Edit"） |
| `--disallowedTools <tools...>` | 禁止的工具列表 |

#### 输入输出

| 参数 | 说明 |
|------|------|
| `-p, --print` | 打印响应后退出（适用于管道） |
| `--bare` | 脚本化 `-p` 调用，跳过 hooks、LSP、插件同步等 |
| `--output-format <format>` | 输出格式：text, json, stream-json（仅 --print） |
| `--input-format <format>` | 输入格式：text, stream-json（仅 --print） |
| `--json-schema <schema>` | 结构化输出验证的 JSON Schema |
| `--include-partial-messages` | 包含部分消息块（仅 stream-json 模式） |
| `--replay-user-messages` | 将用户消息重放到 stdout |
| `--verbose` | 覆盖配置中的详细模式设置 |

#### 系统提示

| 参数 | 说明 |
|------|------|
| `--system-prompt <prompt>` | 使用指定的系统提示 |
| `--append-system-prompt <prompt>` | 追加系统提示到默认提示 |

#### MCP 与插件

| 参数 | 说明 |
|------|------|
| `--mcp-config <configs...>` | 从 JSON 文件或字符串加载 MCP 服务器 |
| `--strict-mcp-config` | 仅使用 --mcp-config 的 MCP 服务器 |
| `--plugin-dir <path>` | 从目录加载插件（可重复指定多个） |
| `--disable-slash-commands` | 禁用所有技能 |

#### 工具控制

| 参数 | 说明 |
|------|------|
| `--tools <tools...>` | 指定可用工具："" 禁用所有, "default" 使用全部, 或指定工具名 |
| `--add-dir <directories...>` | 添加允许工具访问的额外目录 |

#### 工作区与 Git

| 参数 | 说明 |
|------|------|
| `-w, --worktree [name]` | 为会话创建新的 git worktree |
| `--tmux` | 为 worktree 创建 tmux 会话（需要 --worktree） |
| `--from-pr [value]` | 恢复与 PR 关联的会话 |

#### 集成与调试

| 参数 | 说明 |
|------|------|
| `--ide` | 启动时自动连接 IDE |
| `--chrome` | 启用 Claude in Chrome 集成 |
| `--no-chrome` | 禁用 Chrome 集成 |
| `-d, --debug [filter]` | 启用调试模式，可选类别过滤（如 "api,hooks"） |
| `--debug-file <path>` | 将调试日志写入指定文件 |
| `--brief` | 启用 SendUserMessage 工具用于 Agent 与用户通信 |

#### 其他

| 参数 | 说明 |
|------|------|
| `--betas <betas...>` | API 请求中包含的 beta 头（仅 API 密钥用户） |
| `--file <specs...>` | 启动时下载的文件资源，格式：file_id:relative_path |
| `--max-budget-usd <amount>` | API 调用的最大花费金额（仅 --print） |
| `--settings <file-or-json>` | 加载额外设置的 JSON 文件或字符串 |
| `--setting-sources <sources>` | 加载的设置源：user, project, local |

### 子命令

#### `claude auth`

管理认证。

```bash
claude auth login              # 登录（OAuth 或 API Key）
claude auth login --console    # 使用 Anthropic Console (API 计费) 登录
claude auth status             # 查看认证状态
claude auth logout             # 登出
```

#### `claude mcp`

配置和管理 MCP 服务器。

```bash
claude mcp add <name>              # 添加 MCP 服务器
claude mcp remove <name>           # 移除 MCP 服务器
claude mcp list                    # 列出所有 MCP 服务器
claude mcp get <name>              # 获取服务器详情
```

#### `claude plugin` / `claude plugins`

管理 Claude Code 插件。

```bash
claude plugin install <name>       # 安装插件
claude plugin uninstall <name>     # 卸载插件
claude plugin list                 # 列出已安装插件
claude plugin marketplace          # 管理插件市场
```

#### `claude agents`

列出已配置的 Agent。

```bash
claude agents
```

#### `claude install`

安装 Claude Code 原生版本。

```bash
claude install              # 安装稳定版
claude install latest       # 安装最新版
claude install 2.1.80       # 安装指定版本
claude install --force      # 强制重装
```

#### `claude doctor`

检查 Claude Code 自动更新器的健康状态。

```bash
claude doctor
```

#### `claude update` / `claude upgrade`

检查并安装更新。

```bash
claude update
claude upgrade
```

#### `claude setup-token`

设置长期认证令牌（需要 Claude 订阅）。

```bash
claude setup-token
```

### 模型选项

| 模型别名 | 说明 |
|------|------|
| `opus` | Claude Opus 4.6 - 最强模型，复杂任务 |
| `sonnet` | Claude Sonnet 4.6 - 平衡性能和成本（推荐） |
| `haiku` | Claude Haiku 4.5 - 快速响应，简单任务 |

也可使用完整模型名称，如 `claude-opus-4-6-20250929`。

### 使用示例

```bash
# 启动交互式会话
claude

# 非交互式输出（适用于脚本）
claude -p "解释这个函数的作用"

# 指定模型
claude --model opus

# 恢复上次会话
claude -c

# 从特定 PR 恢复
claude --from-pr 123

# 在隔离的 worktree 中工作
claude -w feature-branch

# 使用自定义 MCP 配置
claude --mcp-config ./mcp-servers.json

# 调试模式
claude --debug

# 限制工具访问
claude --tools "Read,Edit,Bash(git:*)"

# 设置权限模式
claude --permission-mode acceptEdits
```
